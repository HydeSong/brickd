## 问题分析

访问 `http://172.20.10.3:8000/components` 显示"页面未找到"，可能的原因：

1. **路由配置问题**：dumi 配置可能没有正确设置组件文档的路径映射
2. **文档结构问题**：文档结构可能不符合 dumi 的要求
3. **预览服务器问题**：预览服务器可能没有使用最新的构建产物

## 解决方案

### 1. 修改 dumi 配置文件

更新 `.dumirc.ts` 配置，确保 dumi 能够正确识别和处理组件文档：

```tsx
import { defineConfig } from 'dumi';

export default defineConfig({
  outputPath: 'docs-dist',
  themeConfig: {
    name: 'brickd',
    nav: [
      {
        title: '指南',
        link: '/guide',
      },
      {
        title: '组件',
        link: '/components',
      },
    ],
    sidebar: {
      '/components': [
        {
          title: '基础组件',
          children: [
            '/components/Button',
            '/components/Icon',
            '/components/Typography',
          ],
        },
        {
          title: '布局组件',
          children: [
            '/components/Layout',
            '/components/Grid',
            '/components/Flex',
            '/components/Space',
          ],
        },
        {
          title: '导航组件',
          children: [
            '/components/Menu',
            '/components/Tabs',
            '/components/Breadcrumb',
            '/components/Pagination',
            '/components/Steps',
            '/components/Anchor',
          ],
        },
      ],
    },
  },
  resolve: {
    docDirs: ['docs', 'src/components'],
  },
  exportStatic: {
    htmlSuffix: true,
  },
});
```

### 2. 检查并更新文档结构

确保 `docs/components.md` 文件存在并且内容正确，作为组件页面的索引页。

### 3. 重新构建和预览

1. 停止当前的预览服务器
2. 重新构建文档：`pnpm run docs:build`
3. 重新启动预览服务器：`pnpm run docs:preview`

### 4. 验证解决方案

访问 `http://172.20.10.3:8000/components` 检查是否能够正确显示组件页面，然后尝试访问具体的组件文档页面，如 `http://172.20.10.3:8000/components/Button`。

## 预期结果

- 能够成功访问 `http://172.20.10.3:8000/components` 页面
- 页面显示组件分类和列表
- 能够点击组件链接进入具体的组件文档页面
